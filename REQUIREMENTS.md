# Meeting Context Hub - 요구사항 명세서

## 개요

팀원들과 함께 사용할 수 있는 웹 애플리케이션으로, 회의록을 PRD/Action Items 형식으로 요약하고, 흩어진 컨텍스트를 자동 태깅하여 옵시디언에 저장하는 시스템

---

## 핵심 기능

### 1. 회의록 요약 자동화

**입력**:
- 1차: 텍스트 (외부 STT 도구로 변환된 내용)
- 2차: 음성 파일 (Whisper API STT)
- 최종: 실시간 녹음

**출력**:
- PRD 형식 (문제/목표/범위/요구사항 구조화)
- Action Items (담당자/데드라인/할일 목록)

### 2. 컨텍스트 그룹핑

**입력 소스**:
- 슬랙 메시지 (API 연동, 읽기 전용)
- 노션 페이지 (API 연동, 읽기 전용)
- 수동 복사-붙여넣기 텍스트
- 회의록 요약 결과

**그룹핑 방식**:
- AI 기반 동적 태그 생성
- 기존 태그 재활용 (유사 주제 감지)
- 프로젝트/작업 단위 자동 분류

**활용**:
- 진행 상황 트래킹 (각 프로젝트/작업의 현재 상태)
- 질문-답변 검색 ("이 프로젝트 어디까지 했지?")

### 3. 옵시디언 연동

**저장소**: 환경변수로 관리되는 옵시디언 볼트 경로
- 현재: `/Users/jaehojang/Documents`

**저장 형식**:
- 마크다운 파일
- YAML frontmatter (태그, 메타데이터)

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | Next.js (React) |
| 백엔드/DB | Supabase (PostgreSQL, Auth, Storage) |
| AI | Claude Code SDK (OAuth Key) |
| 외부 연동 | Slack API, Notion API (읽기 전용) |
| 저장소 | Obsidian Vault (로컬 마크다운) |

---

## 사용자

- **범위**: 팀원들과 함께 사용
- **인증**: Supabase Auth (로그인 필요)

---

## 설계 원칙

- **선언형 구조**: 추상화하여 재사용 가능하게
- **모듈화**: 저장소 레이어 분리 (옵시디언 → 다른 저장소로 교체 가능)
- **점진적 확장**: 텍스트 → STT → 녹음 순차적 개발

## 성공 기준

1. 텍스트 입력 시 PRD 형식 + Action Items가 정확히 추출됨
2. 여러 소스의 컨텍스트가 자동 태깅되어 옵시디언에 마크다운으로 저장됨
3. "이 프로젝트 어디까지 했지?" 질문에 관련 컨텍스트를 기반으로 답변 가능
4. 팀원들이 로그인하여 함께 데이터를 조회/입력 가능
